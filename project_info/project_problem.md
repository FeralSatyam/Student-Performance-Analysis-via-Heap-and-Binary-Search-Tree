# Problem Statements and Proposed Solutions

## Problem 1: Automated Academic Triage for At-Risk Student Identification

**Problem Statement:** In large educational institutions with thousands of students, manually identifying those most vulnerable to academic failure is inefficient and prone to oversight. Without an automated priority system, at-risk students may remain unnoticed until after their performance has substantially declined, limiting the effectiveness of potential interventions. The challenge is to create a system that continuously evaluates all students based on multiple behavioral and academic factors, then instantly surfaces those requiring immediate attention regardless of their position in the enrollment list.

**Solution and Strategy:** This problem will be solved using a **max-heap data structure** with manual heap maintenance algorithms. The strategy involves calculating a composite Risk Factor score for each student using weighted attributes including attendance percentage, hours studied, previous exam scores, tutoring sessions attended, and motivation level. Lower attendance, fewer study hours, lower prior performance, and low motivation increase the risk score. Each student record is stored as a heap node containing their risk score as the priority key alongside their complete academic profile. The heapify algorithm builds the initial priority queue in linear time by performing bottom-up sift-down operations from the last non-leaf node to the root. For extracting the highest-priority students, the sift-down operation restores heap order after removing the root. If student data updates in real-time, the system uses sift-up operations for insertions or marks stale entries for lazy deletion. This approach ensures that extracting the top five most at-risk students requires only logarithmic time per extraction, making it suitable for institutions needing rapid triage decisions without scanning the entire student population.

## Problem 2: Efficient Resource Allocation Through Performance-Based Grouping

**Problem Statement:** Creating effective peer-tutoring partnerships requires quickly identifying students at opposite ends of the academic performance spectrum, specifically those who can serve as tutors (high-performing) and those who need assistance (struggling). Performing this identification through linear searches would require scanning all 6,000+ students separately for each query, which becomes computationally expensive when such groupings are needed frequently or across different score brackets. The challenge is to enable rapid range queries that retrieve all students within arbitrary score intervals without repeatedly traversing the entire dataset.

**Solution and Strategy:** This problem will be solved using a **Binary Search Tree (BST)** with recursive in-order traversal and optimized range query algorithms. The BST stores each student using their exam score as the key, with nodes containing complete student records. The insertion strategy builds the tree with average logarithmic time complexity per student. For range queries such as finding tutors within 90-100 percent or learners within 40-50 percent, a recursive range search algorithm traverses only the subtrees that could potentially contain values within the requested interval—pruning entire branches where all keys fall below the lower bound or above the upper bound. This yields O(k + log n) time complexity where k is the number of students returned, drastically outperforming linear searches when retrieving small subsets. Additionally, a complete in-order traversal of the BST produces a sorted list of all students by exam score, which can be visualized as a distribution histogram for non-technical stakeholders. This dual capability makes the BST strategy effective for both targeted range queries and holistic performance visualization while maintaining efficient operations.
