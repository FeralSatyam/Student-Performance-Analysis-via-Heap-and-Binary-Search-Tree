## 2. Problem 1: Automated Academic Triage (Heap Implementation)

### 2.1 Risk Factor Formula Definition

You must define a formula using available columns. **I recommend:**

```
Risk_Score = 
    (100 - Attendance) * 0.4 +           # Low attendance increases risk
    (10 - Hours_Studied) * 1.5 +         # Low study hours (max 10)
    (100 - Previous_Scores) * 0.3 +      # Low prior performance

```

**Where Motivation_Level_numeric:**
- Low → 10
- Medium → 5
- High → 0

**Normalization:** Risk scores will range roughly 0-100.  
**Higher risk score = more urgent intervention needed.**

### 2.2 Data Structure: Max-Heap (Priority Queue)

**Why max-heap:** You need to extract the highest-risk students first.

**Heap representation:** Use a Python list where index 0 is unused or the root at index 0.

**Operations to implement manually (no heapq):**

| Operation | Algorithm | Time Complexity |
|-----------|-----------|-----------------|
| `heapify(arr)` | Bottom-up: start from last parent (n//2 - 1) and sift-down each | O(n) |
| `push(heap, item)` | Append to end, then sift-up | O(log n) |
| `pop(heap)` | Swap root with last, pop last, sift-down new root | O(log n) |
| `peek(heap)` | Return heap[0] (without removing) | O(1) |

**Item format:** `(risk_score, student_index, student_data)`  
Or store student as a custom object/dictionary.

### 2.3 Sift-Up Algorithm (for insertion)

```
function sift_up(heap, index):
    parent = (index - 1) // 2
    while index > 0 and heap[index][0] > heap[parent][0]:
        swap(heap[index], heap[parent])
        index = parent
        parent = (index - 1) // 2
```

### 2.4 Sift-Down Algorithm (for heapify and pop)

```
function sift_down(heap, index, size):
    largest = index
    left = 2 * index + 1
    right = 2 * index + 2
    
    if left < size and heap[left][0] > heap[largest][0]:
        largest = left
    if right < size and heap[right][0] > heap[largest][0]:
        largest = right
    
    if largest != index:
        swap(heap[index], heap[largest])
        sift_down(heap, largest, size)
```

### 2.5 Heapify from List

```
function heapify(heap):
    n = len(heap)
    for i from n//2 - 1 down to 0:
        sift_down(heap, i, n)
```

## 3. Problem 2: Resource Allocation (BST Range Queries)

### 3.1 BST Node Structure

```
class BSTNode:
    def __init__(self, exam_score, student_data, student_id):
        self.key = exam_score          # Used for ordering
        self.student_id = student_id   # Unique identifier
        self.data = student_data       # Full student record
        self.left = None
        self.right = None
```

### 3.2 BST Operations to Implement

| Operation | Algorithm | Complexity |
|-----------|-----------|------------|
| `insert(root, node)` | Recursive or iterative comparison | O(log n) avg, O(n) worst |
| `search(root, key)` | Standard BST search | O(log n) avg |
| `inorder_traversal(root)` | Recursive left-root-right | O(n) |
| `range_query(root, low, high, result_list)` | Modified recursive search | O(k + log n) |

### 3.3 Range Query Algorithm (Crucial for your problem)

```
function range_query(node, low, high, result):
    if node is None:
        return
    
    # If current node's key is greater than low, search left subtree
    if low < node.key:
        range_query(node.left, low, high, result)
    
    # If current node's key is within range, add it
    if low <= node.key <= high:
        result.append((node.key, node.data))
    
    # If current node's key is less than high, search right subtree
    if node.key < high:
        range_query(node.right, low, high, result)
```

**Why this is efficient:** It avoids traversing subtrees that cannot contain values in the range.

### 3.4 In-Order Traversal for Visualization

```
function inorder_collect(node, result_list):
    if node is None:
        return
    inorder_collect(node.left, result_list)
    result_list.append((node.key, node.data))
    inorder_collect(node.right, result_list)
```

This gives you a **sorted list of all students by exam score** – perfect for distribution visualization.

| Operation | Data Structure | Time | Space |

## 10. Common Pitfalls to Avoid

| Pitfall | Solution |
|---------|----------|
| Forgetting to convert strings to ints | Explicit type conversion for all numeric fields |
| BST becoming skewed | Shuffle data before insertion or implement balancing |
| Heap sift-down off-by-one errors | Test with small heap of 3-5 elements first |
| Memory/stack recursion limit for BST | Use iterative traversal if recursion depth > 1000 (unlikely with 6000 nodes, depth ~13-6000 depending on balance) |
| Intermixing student index with data | Keep a unique ID per student (use list index or add sequential ID during parse) |
